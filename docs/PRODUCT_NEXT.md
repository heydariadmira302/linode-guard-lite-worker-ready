# Linode Guard Lite 下一阶段产品方案

> 状态：需求对齐稿。先按本文定方向，再进入开发。

## 1. 核心开发前提

### 1.1 API / Service 优先

本项目必须继续坚持 API-first / Service-first：

- Telegram 只是展示窗口和交互入口。
- 业务逻辑必须沉到 `src/services/*`、`src/storage/*` 和 HTTP API 层。
- Telegram handler / callback 只负责：
  - 展示中文菜单
  - 收集用户输入
  - 调用 service / API
  - 渲染结果
- 不要把新增业务逻辑直接堆在 Telegram callback 文件里。
- 后续 Web 后台、CLI、Cloudflare Worker API 都应该复用同一套 service 逻辑。

### 1.2 功能拆文件

不要把所有内容挤在一个大文件里。

推荐按功能拆分：

```text
src/services/group-service.ts
src/services/account-service.ts
src/services/instance-service.ts
src/services/security-service.ts
src/services/admin-presence-service.ts
src/services/schedule-service.ts

src/api/groups.ts
src/api/accounts.ts
src/api/instances.ts
src/api/security.ts
src/api/admin-presence.ts
src/api/schedules.ts

src/telegram/group-renderer.ts
src/telegram/account-flow.ts
src/telegram/instance-renderer.ts
src/telegram/security-renderer.ts
src/telegram/admin-presence-renderer.ts
src/telegram/schedule-renderer.ts
src/telegram/keyboards.ts
```

原则：一个文件只负责一个清晰功能。

### 1.3 中文优先

Telegram 里的按钮、菜单、提示、错误原因必须尽量中文描述。

不要出现用户看不懂的英文按钮，例如：

```text
boot
shutdown
delete
checkin
schedule
```

应该显示为：

```text
开机
关机
删除
打卡
定时任务
```

内部 callback_data 可以继续用英文机器码，但用户可见文本必须中文。

## 2. 已确认产品规则

### 2.1 分组规则

- 一个账号只属于一个分组。
- 一个分组可以包含多个账号。
- 默认分组为：`未分组`。
- 分组下可以查看账号和服务器。
- 分组后续要支持批量操作。

### 2.2 安全事件基线

添加账号时必须建立安全基线：

- 历史登录记录不通知。
- 从添加 key 之后的新登录开始通知。
- 添加账号时记录：
  - `security_baseline_at`
  - 当前最新 login id / cursor（如 Linode API 可用）

### 2.3 Telegram 键盘规则

固定 Reply Keyboard：

```text
🏠 主菜单 | ❤️ 打卡
🖥 服务器 | 👤 账号
```

Inline Keyboard 用于当前页面操作，例如：

```text
详情
开机
关机
删除
确认
取消
下一页
返回
```

### 2.4 保活策略规则

- 保活策略以用户设置为准。
- 保活策略规则大于一切：已经配置并启用的保活策略，到触发时间后直接执行，不需要再次二次确认。
- 保活策略触发的自动批量删机优先级最高。
- 其他高危功能需要二次确认。
- 设置“自动删除所有服务器”这个策略本身必须给出明显高危提示，但不要求额外文本二次确认。

### 2.5 服务器 IP 展示

服务器列表和详情需要展示 IPv4。

- 列表显示第一个 IPv4。
- 详情显示全部 IPv4。
- IPv6 暂不展示。

## 4. 账号管理目标

### 4.1 添加账号流程

目标流程：

```text
账号管理 -> 添加账号
1. 输入账号昵称
2. 选择分组 / 新建分组 / 未分组
3. 输入 Linode Token
4. 自动检测 Token 是否正确可用
5. 建立安全基线
6. 添加成功通知
```

### 4.2 账号昵称

账号昵称必须支持中文，例如：

```text
西班牙1
日本备用
洛杉矶主号
```

建议规则：

- 长度 1-32。
- 支持中文、英文、数字、空格、下划线、短横线。
- 禁止空昵称。
- 禁止重复昵称。
- 内部逻辑不要依赖昵称作为唯一稳定 ID。

### 4.3 Token 检测

添加账号时必须检测 key 是否正确能用。

建议检测：

```http
GET /account
GET /linode/instances
```

失败提示必须中文化：

```text
Token 无效：请检查是否复制完整。
权限不足：请确认 Token 有读取账号和实例权限。
Linode API 暂时不可用：请稍后重试。
```

成功提示：

```text
✅ 账号添加成功

账号：西班牙1
分组：西班牙
Token 状态：可用
服务器数量：3
安全基线：已建立，历史登录不通知
```

## 5. 分组功能目标

### 5.1 基础功能

需要支持：

- 新建分组
- 重命名分组
- 删除空分组
- 查看分组下账号
- 查看分组下服务器
- 添加账号时选择分组
- 移动账号到其他分组

### 5.2 分组限制

- 账号只能属于一个分组。
- 删除非空分组时，不直接删除，提示先移动或删除账号。

## 6. 服务器功能目标

### 6.1 服务器入口

点击“服务器”不能是空壳，必须可操作。

菜单：

```text
🖥 服务器管理

请选择查看范围：

全部服务器
按分组查看
按账号查看
运行中
已关机
```

### 6.2 服务器列表

列表展示：

```text
🟢 web-01
账号：西班牙1
分组：西班牙
地区：es-mad
IPv4：1.2.3.4
状态：运行中
```

### 6.3 服务器详情

详情展示：

```text
🖥 服务器详情

名称：web-01
ID：123456
账号：西班牙1
分组：西班牙
地区：es-mad
状态：运行中
IPv4：
- 1.2.3.4
- 5.6.7.8
```

IPv6 暂不展示。

### 6.4 服务器操作

根据状态展示按钮：

- 运行中：关机、重启、删除
- 已关机：开机、删除
- 未知状态：刷新、详情

除保活策略自动触发外，高危操作需要二次确认：

- 删除服务器
- 批量删除服务器
- 批量关机/重启也建议二次确认

## 7. 安全事件目标

### 7.1 历史事件处理

添加 key 以前的历史登录不通知。

实现要求：

- 添加账号成功时建立安全基线。
- 后续只通知基线之后的新登录。

### 7.2 安全事件通知

示例：

```text
⚠️ 新登录事件

账号：西班牙1
用户：root
IP：1.2.3.4
时间：2026-05-15 17:20

这是你本人操作吗？
```

按钮：

```text
是我
不是我
❤️ 打卡
```

## 8. 定时任务目标

Telegram 内必须能按钮化新增、查看、暂停、删除定时任务。

菜单：

```text
⏰ 定时任务

新增任务
任务列表
暂停全部
返回主菜单
```

新增任务目标流程：

```text
选择动作：开机 / 关机 / 重启
选择范围：全部账号 / 分组 / 账号 / 单台服务器
选择时间：每天 08:00 / 每天 22:00 / 自定义 Cron
保存
```

当前已先落地 Telegram 快速新增 MVP：开机 / 关机 + 全部账号或单账号 + 每天 08:00 / 每天 22:00，也支持输入 `09:30` 这类每天固定时间或 5 段自定义 Cron。后续再补分组、单台服务器和重启。

## 9. 保活打卡目标

### 9.1 固定打卡入口

Reply Keyboard 固定提供：

```text
❤️ 打卡
```

每条重要消息下方也尽量提供 Inline 打卡按钮。

### 9.2 保活策略配置

菜单：

```text
❤️ 保活打卡

当前状态：
上次打卡：2 小时前
提醒时间：12 小时
最终动作时间：24 小时
最终动作：删除所有服务器
作用范围：全部账号 / 单账号 / 分组

立即打卡
设置提醒时间
设置最终动作时间
设置最终动作
设置作用范围（全部账号 / 单账号 / 分组）
暂停保活策略
```

### 9.3 自动批量删机规则

- 用户设置了保活自动删机策略后，到触发时间自动执行。
- 这个自动动作不再二次确认。
- 设置策略时必须明确警告风险。
- 执行后必须写审计日志并通知管理员。
- 最终动作必须按策略作用范围执行：全部账号、单账号或分组；不能因为策略触发而误操作范围外账号。

## 10. 开发优先级建议

### Phase 1：Telegram 可用性和账号体验

- Reply Keyboard 固定入口。
- 主菜单中文化和美化。
- 服务器按钮真正可用，显示 IPv4。
- 添加账号支持中文昵称。
- 添加账号检测 Token。
- 添加账号成功提示。
- 添加账号时建立安全基线。

### Phase 2：分组

- 新增 groups 表。
- 账号归属分组。
- 分组菜单。
- 按分组查看账号/服务器。
- 移动账号分组。

### Phase 3：安全事件优化

- 基线之后才通知。
- 安全事件菜单中文化。
- 是我 / 不是我 / 打卡按钮。

### Phase 4：定时任务和保活策略按钮化

- Telegram 新增/查看/暂停/删除定时任务。
- 保活策略完整配置。
- 自动删机策略按用户设置执行。
